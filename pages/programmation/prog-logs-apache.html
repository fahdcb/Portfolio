CONTEXTE

Comprendre le trafic d'un serveur web

Un webmaster ou administrateur système a besoin de savoir qui se connecte à son serveur, quand, et combien de fois — c'est le rôle des fichiers de logs. Ce projet consistait à développer en langage C un programme qui lit et analyse les logs d'un serveur Apache, puis restitue les statistiques sous trois formes différentes : interface web (CGI), fenêtre graphique, et terminal texte.

STRUCTURE

Organisation du projet livré

  • Fichiers sources .c et .h dans RT1SA05/src/
  • Makefile fonctionnel (compilation + installation automatique)
  • Fichiers HTML/CSS pour l'interface web
  • Rapport PDF avec algorithmes en pseudo-code, structures de données et captures
  • Archive .tgz correctement nommée et soumise sur eCampus

FONCTIONNEMENT

Les trois interfaces du programme

🌐

Interface Web

Le programme se comporte comme un CGI-BIN : il génère du code HTML avec printf(), lu et interprété par Apache. L'histogramme s'affiche dans le navigateur.

mode par défaut
📊

Interface Graphique

Une fenêtre s'ouvre avec libgraphique et affiche les barres de l'histogramme avec les valeurs en pourcentage par mois.

-gr
💻

Interface Texte

Le programme affiche directement dans le terminal le nombre total de connexions et la répartition mensuelle en pourcentage.

-txt
OBJECTIFS

Ce qu'on devait réaliser

  • 1 Parser le fichier /var/log/apache2/access.log au format Common Log Format (CLF)
  • 2 Calculer le nombre total de requêtes et leur distribution mensuelle en pourcentage
  • 3 Proposer trois modes d'affichage : web (CGI), graphique (histogramme) et texte (terminal)
  • 4 Mettre en place un Proof of Concept complet et déployable sur un serveur Apache réel
